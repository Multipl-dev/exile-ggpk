# LESSONS | Lessons Learned Log
<!-- Type: Append-only | Scope: Project-wide | Update: Per incident -->

---

## Purpose

Document lessons learned from errors, incidents, and discoveries to prevent recurrence.

---

## Template

```markdown
### LESSON: [Short Name]
- **Date:** YYYY-MM-DD
- **Category:** Bug | Performance | Architecture | Process
- **Severity:** Minor | Major | Critical
- **Recurrence Risk:** Low | Medium | High
- **Agents:** [Applicable agents: nav, arch, val, debug, etc.]
- **What happened:** [Description]
- **Root cause:** [Why it happened]
- **Prevention:** [How to avoid in future]
- **Applied to:** [Files/processes updated]
```

### Field Definitions

| Field | Values | Meaning |
|-------|--------|---------|
| Severity | Minor | Inconvenience, quick fix |
| | Major | Significant time loss, partial revert |
| | Critical | Full revert, data loss, or blocking |
| Recurrence Risk | Low | Unique circumstances, unlikely to repeat |
| | Medium | Could happen again under similar conditions |
| | High | Will definitely recur without active prevention |
| Agents | nav, arch, val, etc. | Which agents should surface this lesson |
